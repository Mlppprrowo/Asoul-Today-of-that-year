# Asoul-Today-of-that-year
一个简单的asoul动态那年今日推送小系统


🌟 A-SOUL 动态追踪系统 (Today of That Year)

这是一个基于 GitHub Actions 的手动触发式爬虫系统。它不仅能为你监控 A-SOUL 成员的 B站动态，还自带了“历史数据库”，让你在 Fork 后的第一次运行就能享受极速体验。

🚀 为什么选择本项目？

前人栽树，后人乘凉：本仓库已内置了 A-SOUL 成员的大量历史动态数据库（dynamic.db）。你 Fork 之后，系统会自动识别这些旧动态，第一次运行即是“增量更新”，无需从几年前的数据开始漫长等待。

隐私隔离：虽然数据库是共享的，但你的推送地址和登录凭证（Secrets）是完全私密的，不会泄露给原作者或其他用户。

零成本维护：无需购买服务器，利用 GitHub 提供的免费算力，按需手动运行。

🛠️ 快速开始 (Fork & Run)

1. 点击 Fork

点击页面右上角的 Fork 按钮，将本项目克隆到你自己的账号下。

2. 获取你的专属密钥

B站 Cookie：登录 Bilibili 后，按 F12 在网络请求的 Headers 中找到 Cookie 并复制。

机器人提醒地址：本项目支持多种推送方式。

钉钉机器人：在群设置中添加“自定义机器人”，设置关键词，复制生成的 Webhook 地址。

其他机器人：你也可以选择使用 企业微信机器人、飞书机器人 或 Server酱 等其他支持 Webhook 的提醒地址，只需将对应的 API 链接填入即可。

3. 配置 GitHub Secrets

在你 自己的 仓库中，点击 Settings -> Secrets and variables -> Actions。

点击 New repository secret，分别添加以下两个变量：

BI_COOKIE：填入你的 B 站 Cookie。

PUSH_URL：填入你选择的机器人（钉钉/企微/飞书等）的 Webhook 完整链接。

▶️ 如何手动触发

由于本项目目前设定为手动触发（保护账号安全），你需要手动点火：

进入你仓库顶部的 Actions 页面。

在左侧点击工作流名称（如 AsoulDaily）。

点击右侧的 Run workflow 下拉框，然后点击绿色按钮。

极速体验：因为仓库里已有历史数据，第一次运行通常仅需 1 分钟 左右即可完成推送。

🔮 未来功能预告

本项目仍在持续迭代中，以下功能将在未来的版本中加入：

定时发布消息插件：我们将引入自动化的定时任务插件，让你无需手动点击，即可在每天固定时间收到动态汇总。

🗃️ 数据库说明

dynamic.db：核心数据库文件。每次运行成功后，系统会自动更新新抓取的动态并提交回你的仓库。

安全性：数据库内仅存储动态文本和图片链接，绝对不会存储你的 Cookie 或任何推送密钥。

💡 开发建议

如果你想修改监控的成员名单，只需编辑 main.py 中的 members 字典，填入对应的 B 站 UID 即可。
